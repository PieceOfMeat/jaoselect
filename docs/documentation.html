Usage

$('#myselectid').jaoselect(<options>);
Settings list
name default description
dropdown true boolean value, indicating appearance of widget: combobox drop-down list or plain options list
multiple get from initial <select> element "multiple" attribute boolean value, if set to true, options will be presented by checkboxes, false - options will be presented as radio buttons, providing single selection
width get from initial <select> element combobox header width if dropdown == true, or options list width if dropdown == false
height get from initial <select> element combobox header height if dropdown == true, or options list height if dropdown == false
maxDropdownWidth 800 makes sence only if dropdown == true, specifies max width of dropdown options list
maxDropdownHeight 400 makes sence only if dropdown == true, specifies max height of dropdown options list
placeholder &nbsp; text to be displayed when no value is selected
Templating functions
template.placeholder $.fn.jaoselect.template.placeholder (settings.placeholder) Templating function which is called when no value is selected.
template.singleValue $.fn.jaoselect.template.singleValue (item text and image if exists) Templating function which is called when exactly one value is selected
template.multipleValue $.fn.jaoselect.template.selectedCount (text "Selected %N of %M") Templating function which is called when selected more than one value
Note that you can override default settings for whole plugin: $.fn.jaoselect.defaults.dropdown = false; $.fn.jaoselect.defaults.template.placeholder = function() {return 'boo!'};

Templating functions

When user select/deselect any option in options list, plugin updates header of combobox. Depending on quantity of selected items we distinguish three separate cases of templating (as set in plugin settings): 0: template.placeholder (calls with no arguments)
1: template.singleValue (calls with one argument representing one value)
>=1: template.multipleValue (calls with one argument - array of values)
This functions run in context of current JaoSelect object, so they get full access to JaoSelect settings and internal keys.
When one or more values selected, functions receive them as argument. Each value is represented by plain JS object containing full info about item: { title: Item title value: Item value, image: path to item image, cls: class name of initial <option> element } It will also contain any data you add to initial <option> element by html5 data-attributes! Plugin declares four templating functions: $.fn.jaoselect.template.placeholder returns span with current settings.placeholder text
$.fn.jaoselect.template.singleValue returns span with item title and image, if exists
$.fn.jaoselect.template.multipleValue returns spans with item titles and images, if exists, comma-separated
$.fn.jaoselect.template.selectedCount returns text 'Selected %N of %M'
But you are free to extend $.fn.jaoselect.template with your own functions. This is very flexible way to style your comboboxes! Example: $.fn.jaoselect.template.multipleValueWithClipping = function(values) { var i, html = []; // Adding first two values as we do in singleValue // Note that is we want to use other templating functions? we must them in correct context! html.push(this.settings.template.singleValue.call(this, values[0])); html.push(this.settings.template.singleValue.call(this, values[1])); if (values.length > 2) { html.push('and ' + (values.length - 2) + ' more...'); } return html.join(', '); }
